Run a Webserver Anywhere

Being able to run a webserver anywhere on my development system has become one of my essential developer tricks. Not only to run projects I'm working on, but just to spontaneously run projects/templates as I look around the system. Of course, I am largely talking about html/js browser base projects as I do a lot of static site development. Running php, asp.net or even express projects is a little more restrictive.

With Apache or IIS you're largely restricted to the folders they are configured to run sites from. So typically you develop somewhere and then copy the output to the server location.

Node.js has a number of lightweight webserver packages that can be installed globally. When a package is installed globally it can usually be run anywhere on the system via the command line. So this means that if one of Node's http server packages is installed globally, it will run anywhere on the system.

The package I use is the live-server package. I have used the http-server but switched to live-server as it has the live reload ability built in.

Once the package is installed globally with the npm install -g command, navigate to a project root on your system and run the package via the command line.

You will notice that the server attempts to run on port 8080. If this port is being used, it trys another one. What this means is that you can run more than one server/site on a system and they will simply run from a different port. The web servers themselves have a small memory footprint, so there's not really a penalty to running more than one instance.

Add To Context Menu

Navigating the file system via the command line is a bit of a pain. Fortunately, opening the command line within a folder is available by default as a context menu option in explorer - although you will probably have to hold the shift key to see it.

Doing it this way you only need to type in "live-server". Simple enough.

But to make it even easier, I add a "Run Live Server" menu option to the explorer context menu. This involves a bit of registry editing.

  • Open up the registry editor.
  • Navigate to \HKEY_LOCAL_MACHINE\SOFTWARE\Classes\Directory\background\.
  • Right Click on shell.
  • Select New > Key.
  • Name the key what you want the context menu item to say. Something like "Run Live Server".
  • Underneath your new key, create anothor one and call it command.
  • Modify the value of the command key by right clicking (default) in the right side panel and selecting modify from the menu.

  • Edit the value to run the command line command. In my case cmd.exe /k live-server. I add the /k modifier so that the command line window doesn't close after running.

Open explorer and you should see your new context menu item.

You now have a run anywhere webserver.
